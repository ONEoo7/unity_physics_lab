# Unity Physics Lab

A cross-platform Unity 6.3 LTS project — a hub room with multiple physics experiment
stations. The first experiment is a vibrating Chladni plate where salt grains drift
toward nodal lines, producing the classic interference patterns.

## Project layout

```
Assets/
  InputSystem_Actions.inputactions   shared input map (Player + UI)
  Settings/                          URP render pipeline assets (PC + Mobile)
  _Project/
    Scripts/
      Runtime/                       PhysicsLab.Runtime asmdef (Core, Framework, UI)
      Editor/                        PhysicsLab.Editor asmdef (scaffolder)
    Experiments/
      Chladni/                       PhysicsLab.Chladni asmdef (sim, audio, controller)
    Scenes/                          generated by the scaffolder
    Art/                             generated materials and meshes
    Settings/                        generated ExperimentDefinitions
```

## First-time bring-up

1. Open the project in Unity 6.3 LTS (currently 6000.3.15f1).
2. Let Unity import packages — Burst will compile the salt simulation on first run.
3. In the menu, run **Tools → Physics Lab → Scaffold Scenes**. This creates the
   `Lab` hub and `Chladni` experiment scene, the `ChladniDefinition`
   ScriptableObject, supporting materials and meshes, and adds both scenes to
   the build settings.
4. Open `Assets/_Project/Scenes/Lab.unity`, press Play.
5. WASD + mouse to walk. Look at the blue pedestal and press **E** to enter
   the experiment. **Esc** (or the Back button) returns to the lab.

## Chladni experiment notes

- Frequency slider sweeps 100–1850 Hz. The simulator picks the two surrounding
  named modes from the table in `ChladniExperiment` and linearly blends them.
- Salt grains follow the negative gradient of |u|² of the blended mode shape,
  with a jitter scaled by local amplitude (so settled grains stay put).
- The simulation runs as a Burst-compiled `IJobParallelFor`. Particle count is
  capped at 5,000 on WebGL/Android/iOS and low-VRAM devices, 20,000 otherwise.
- A pure-tone sine driver is generated on the audio thread via
  `OnAudioFilterRead`, clamped to a safe amplitude. Audio defaults to off.

## Adding a new experiment

1. Create a new asmdef under `Assets/_Project/Experiments/<Name>` that references
   `PhysicsLab.Runtime`.
2. Write your experiment's root MonoBehaviour deriving from `ExperimentBase`.
3. Create an `ExperimentDefinition` ScriptableObject pointing at the new scene's
   name, and assign it to an `ExperimentStation` pedestal in the `Lab` scene.
4. Add the new scene to the Build Settings (via the scaffolder menu or manually).
